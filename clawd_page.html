How to Set Up OpenClaw on a Mac Mini (From Zero)
This is a no-fluff walkthrough for getting OpenClaw running on a fresh Mac Mini. Works for complete beginners and anyone in between. ## Prerequisites - A Mac Mini (Apple Silicon) - An Anthropic API key (or other supported provider) - Telegram installed on your phone ## 1. Get Chrome and Claude Download Chrome using Safari. Open
https://claude.ai/new
you'll use this as your lifeline if anything breaks. ## 2. Open Terminal Command+Space type "terminal" hit Enter. Right-click the dock icon Keep in Dock. You'll live here for a bit. Pro tip: The default light terminal theme is unreadable. Go to Terminal Settings Profile and pick a dark theme before you do anything else. Better yet, install iTerm2 later. **Before you continue:** Go to
https://claude.ai/new
and tell Claude "I want to install OpenClaw on my Mac Mini." Claude will guide you through most of the steps below. The workflow is: paste commands Claude gives you into Terminal, then paste any errors back to Claude. Think of Claude as your copilot the steps below are your backup documentation, but Claude can adapt to whatever issues you hit. ## 3. Install Homebrew ```bash /bin/bash -c "$(curl -fsSL
https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh
)" ``` This also installs Xcode Command Line Tools. Takes a few minutes. Important: After installing anything via Homebrew, close the terminal and open a new one before trying to use it. The shell needs to reload its PATH. ## 4. Install Node.js ```bash brew install node@22 ``` Then add it to your PATH: ```bash echo 'export PATH="/opt/homebrew/opt/node@22/bin:$PATH"' >> ~/.zshrc ``` Close terminal, open a new one, verify: ```bash node --version # v22.x.x ``` (Again, if you have any problems or questions, send them to the
claude.ai
chat and iterate.) Optional but recommended: `brew install htop` for process monitoring. ## 5. Install OpenClaw ```bash curl -fsSL
https://openclaw.ai/install.sh
| bash ``` The installer will: - Ask you to acknowledge some risk warnings say yes - Offer Quickstart vs Manual choose Quickstart - Ask which model provider pick yours (e.g., Anthropic) ## 6. Set Up Your API Key (Anthropic) If you chose Anthropic, you need Claude Code installed first: ```bash npm install -g
@anthropic
-ai/claude-code ``` Then run: ```bash claude setup-token ``` This opens a browser for authentication. After logging in, you'll get a long API key (sk-ant-...). Watch out: The terminal may word-wrap the key across multiple lines. Copy it into a text editor first and make sure it's a single unbroken line before pasting it back. ## 7. Set Up Telegram 1. Open Telegram and search for
@BotFather
2. Send it a message, hit "Create New Bot" 3. Copy the bot token it gives you (format: 85040xxxxx:XXXXXXXXX...) 4. Paste it into the OpenClaw installer when it asks for the Telegram bot token ## 8. Configure Skills and APIs The installer will ask about various skills and API keys: - GitHub if you want repo management - OpenAI API key useful for image generation (sk-svcacct-...) - Whisper speech-to-text, optional - Skip anything you don't need (Google Places, etc.) Enable session memory when it asks about hooks. ## 9. OS Permissions macOS will start asking for permissions (local network, etc.). Allow everything you're on a dedicated machine. ## 10. Launch It ```bash openclaw tui ``` Reminder: Tab completion works. Type `openc` + Tab to autocomplete. ## 11. Finish Telegram Pairing Go to your bot's Telegram chat and send any message. It'll respond with your Telegram user ID + a pairing code. Paste the code back to the bot. Now you can control your agent from your phone. ## 12. First Things to Do - Give it a few tasks that trigger OS permission dialogs - Once permissions are sorted, switch to Telegram for everything - Set up accounts it needs (email, GitHub, etc.) - Tell it to save important things to memory - Use /new in the TUI frequently to keep context clean ## Troubleshooting When something breaks (and it will), here's the workflow: **First:** Try restarting the gateway: ```bash openclaw gateway restart ``` **If that doesn't fix it:** Run the doctor: ```bash openclaw doctor --fix ``` **Still broken?** Go back to
https://claude.ai/new
and tell Claude what you tried and what's happening. Claude will probably ask for logs. Here's how to get them: ```bash cat ~/Library/Logs/openclaw/gateway.log | tail -100 cat ~/Library/Logs/openclaw/gateway.err.log | tail -100 ``` **Lost your memory?** (Usually happens when changing agent configs.) Reindex: ```bash openclaw memory index ``` **The key:** Get the agent talking again first, then explain what needs fixing. When it does the right thing, tell it to save that to memory. Context windows fill up use `/new` in the TUI or start fresh sessions in Telegram for best results. Each new task should get a clean slate. ## The Honest Truth OpenClaw is simultaneously vibe-coded slop and an incredible piece of software. Your perception will oscillate between these two constantly. Be patient with it. Keep the context clean. Help it fix its own problems. Remember the whole thing is just a wrapper around the model, and you can inspect and change every part of it. If you break something badly,
claude.ai/new
is your friend. Don't panic.
